CSS Styling (Background, Text, Fonts)

Background Properties

  • background-color: Sets solid colors using color names, HEX, RGB, or HSL.
  • background-image: Embeds an image file using url('image.jpg').
  • background-repeat: Prevents or allows tiling (repeat, no-repeat, repeat-x).
  • background-position: Aligns background graphic locations (center, top right).

Text Formatting

  • color: Controls foreground text colors.
  • text-align: Sets horizontal alignment (left, right, center, justify).
  • text-decoration: Attaches or removes lines (none, underline, line-through).
  • text-transform: Forces case conversions (uppercase, lowercase, capitalize).
  • letter-spacing / word-spacing: Adjusts text spacing intervals.

Controlling Fonts

  • font-family: Establishes font choice with fallback stacks.
  • font-size: Defines scale (px, em, rem, %).
  • font-style: Toggles slant variations (normal, italic).
  • font-weight: Adjusts structural boldness (normal, bold, 100-900).

📦 1. Block Elements and Objects

Definition

  • Block Elements automatically start on a new line and stretch horizontally to fill the entire width of their parent container.
  • Inline Elements sit side-by-side on the same line and only take up as much horizontal space as their content requires.
  • Layout Control over these elements is managed using the CSS display property to change their default browser behavior.
  • Inline-Block Hybrid lets elements sit on the same line but still honors your explicit width, height, margin, and padding settings.
  • Object Embedding utilizes <object> or <iframe> HTML tags to insert external documents, external webpages, or external media assets.

Syntax

/* Changing natural display behaviors */
selector {
  display: block | inline | inline-block | none;
}

Example

<!-- HTML Structure -->
<span class="make-block">I act like a block element now.</span>
<iframe src="https://example.com" width="300" height="200"></iframe>

<!-- Associated CSS -->
<style>
.make-block {
  display: block;
  background-color: lightblue;
  width: 100%;
}
</style>

📋 2. Lists and Tables

Definition

  • Marker Modification is handled by changing the list-style-type property to remove or alter list bullet and number designs.
  • Navigation Pre-requisite requires removing default list padding, list margins, and list bullets to prepare lists for layout styling.
  • Border Collapse combines separated double table borders into clean, single-line borders using border-collapse: collapse;.
  • Data Spacing relies on applying padding to table header (<th>) and table cell (<td>) elements for high readability.
  • Zebra Striping uses structural pseudo-classes to automatically alternate background colors on alternating data rows down the table.

Syntax

/* Removing bullets, collapsing borders, and targeting alternate rows */
ul-selector {
  list-style-type: none;
}
table-selector {
  border-collapse: collapse;
}
row-selector:nth-child(even) {
  background-color: color-value;
}

Example

<!-- HTML Structure -->
<table>
  <tr><th>Item</th></tr>
  <tr><td>Apples</td></tr>
  <tr><td>Oranges</td></tr>
</table>

<!-- Associated CSS -->
<style>
table {
  border-collapse: collapse;
  width: 100%;
}
th, td {
  border: 1px solid black;
  padding: 8px;
}
tr:nth-child(even) {
  background-color: #f2f2f2;
}
</style>

🏷️ 3. CSS ID and Class

Definition

  • Class Selectors are built for wide reusability and can be applied to many different HTML elements on a single page.
  • ID Selectors are strict, unique identity anchors meant to target exactly one specific HTML element per webpage.
  • JavaScript Hooking frequently relies on unique ID names to find and manipulate elements dynamically inside scripts.
  • Specificity Rules state that the browser prioritizes ID style declarations over class style declarations every single time.
  • Selector Symbols differ fundamentally, requiring a dot prefix for naming classes and a hashtag prefix for naming IDs.

Syntax

/* Selecting by class (dot) and selecting by ID (hashtag) */
.class-name {
  property: value;
}
#id-name {
  property: value;
}

Example

<!-- HTML Structure -->
<div id="hero-section">
  <p class="alert-text">Danger! Check your input fields.</p>
  <p class="alert-text">Warning! Connection is slow.</p>
</div>

<!-- Associated CSS -->
<style>
.alert-text {
  color: orange;
  font-weight: bold;
}
#hero-section {
  background-color: #eee;
  padding: 20px;
}
</style>

📦 4. The CSS Box Model

Definition

  • The Core Content forms the innermost box area where text strings, images, and child nodes are rendered.
  • The Padding Layer creates clean, internal breathing room directly around your content, sitting inside your border boundaries.
  • The Border Frame wraps tightly around the padding layer, acting as a visible outline or an invisible structural boundary line.
  • The Margin Boundary generates transparent external space around your element, pushing adjacent page elements away from its edges.
  • The Box-Sizing Fix forces padding and borders inside the declared element width, preventing layouts from breaking out of alignment.

Syntax

/* Standard modern layout initialization sizing rule */
selector {
  box-sizing: border-box;
  padding: top right bottom left;
  margin: top right bottom left;
}

Example

<!-- HTML Structure -->
<div class="box-card">Box Model Content</div>

<!-- Associated CSS -->
<style>
.box-card {
  box-sizing: border-box;
  width: 200px;
  padding: 20px;
  border: 5px solid black;
  margin: 15px;
}
</style>

🗺️ 5. Navigation Bars

Definition

  • Semantic Wrapping isolates main site navigation frameworks using the structural, accessible HTML5 <nav> parent container tag.
  • Structural Foundation uses unordered lists (<ul>) and list items (<li>) to group hyperlink nodes semantically for screen readers.
  • Horizontal Alignment strips block-level layout limits from list items using flex properties or inline display declarations.
  • Interactive States leverage link pseudo-classes to styling link presentation based on past visitor actions or active movements.
  • Clickable Area Expansion converts link anchors into block display setups so users can click anywhere inside the link box padding.

Syntax

/* Flexbox navigation bar line-up pattern */
nav-list-selector { display: flex; list-style-type: none; }
link-selector:hover { property: value; }

Example

<!-- HTML Structure -->
<nav>
  <ul>
    <li><a href="#">Home</a></li>
    <li><a href="#">About</a></li>
  </ul>
</nav>

<!-- Associated CSS -->
<style>
  nav ul { display: flex; list-style-type: none; padding: 0; }
  nav a { display: block; padding: 10px 20px; text-decoration: none; }
  nav a:hover { background-color: #ddd; }
</style>

🎨 6. CSS Colors

Definition

  • Hexadecimal Codes express color channels using combinations of numbers and letters representing Red, Green, and Blue values.
  • RGB Notation declares exact numerical color intensities on a standard scale ranging directly from 0 to 255.
  • The Alpha Channel controls color transparency values on a fractional scale moving between 0.0 and 1.0.
  • HSL Systems organize color selections by mapping out Hue degrees, Saturation percentages, and Lightness percentages logically.
  • Opacity Inheritance differs because the opacity property affects entire elements, while alpha channels only affect target background colors.

Syntax

/* Four alternative methods for defining identical or modified colors */
selector { color: #ff0000; }                  /* Hex */
selector { color: rgb(255, 0, 0); }           /* RGB */
selector { color: rgba(255, 0, 0, 0.5); }     /* RGBA */
selector { color: hsl(0, 100%, 50%); }        /* HSL */

Use code with caution.

Example

<!-- HTML Structure -->
<div class="solid-box">Solid Background Text</div>
<div class="trans-box">Semi-transparent Background Text</div>

<!-- Associated CSS -->
<style>
  .solid-box { background-color: #00ff00; color: black; }
  .trans-box { background-color: rgba(0, 255, 0, 0.3); color: black; }
</style>

📐 7. Page Layout and Site Design

Definition

  • Semantic Page Wrappers organize global code spaces layout structures cleanly using <header>, <main>, and <footer> tags.
  • Flexbox Frameworks orchestrate layout spaces along a single dimension, organizing items smoothly across a row or column track.
  • CSS Grid Engines manage advanced layouts across two dimensions simultaneously, controlling overlapping rows and columns efficiently.
  • Accessibility Enhancements occur naturally when layout semantics allow search engines and assistive engines to crawl layouts accurately.
  • Responsive Control scales components smoothly using layout tools to reorder and adapt elements dynamically to varying viewports.

Syntax

/* One-dimensional Flex layout and two-dimensional Grid layout triggers */
flex-container { display: flex; justify-content: space-between; }
grid-container { display: grid; grid-template-columns: repeat(3, 1fr); }

Use code with caution.

Example

<!-- HTML Structure -->
<div class="layout-grid">
  <header>Header Area</header>
  <main>Main Content Area</main>
  <footer>Footer Area</footer>
</div>

<!-- Associated CSS -->
<style>
  .layout-grid {
    display: grid;
    grid-template-columns: 1fr;
    grid-gap: 10px;
  }
  header, main, footer { background-color: #ccc; padding: 20px; }
